# 그래프 알고리즘 (Graph Algorithms)

## 개요

그래프 알고리즘은 **노드(정점)와 간선으로 이루어진 그래프 구조를 탐색하고 분석하는 알고리즘**입니다. 네트워크, 지도, 소셜 미디어 등 다양한 실생활 문제를 해결하는 데 사용됩니다.

---

## 주요 그래프 알고리즘

### 1. 탐색 알고리즘 (Search Algorithms)

#### 너비 우선 탐색 (BFS - Breadth-First Search)
- **특징**: 시작점에서 가까운 노드부터 차례로 탐색
- **활용**: 최단 경로 찾기, 레벨 탐색
- **시간복잡도**: O(V + E)
- **공간복잡도**: O(V)
- **자료구조**: Queue 사용

#### 깊이 우선 탐색 (DFS - Depth-First Search)
- **특징**: 한 경로를 끝까지 탐색한 후 다른 경로 탐색
- **활용**: 위상정렬, 사이클 감지, 백트래킹
- **시간복잡도**: O(V + E)
- **공간복잡도**: O(V)
- **자료구조**: Stack 사용 (또는 재귀)

---

### 2. 최단 경로 알고리즘 (Shortest Path Algorithms)

#### 다익스트라 (Dijkstra's Algorithm)
- **특징**: 한 노드에서 다른 모든 노드까지의 최단거리 계산
- **제약**: 가중치가 양수일 때만 사용
- **시간복잡도**: O((V + E) log V) - 우선순위 큐 사용 시
- **활용**: GPS 네비게이션, 네트워크 라우팅

#### 벨만-포드 (Bellman-Ford Algorithm)
- **특징**: 음수 가중치도 처리 가능
- **제약**: 음수 사이클 감지 필요
- **시간복잡도**: O(VE)
- **활용**: 금융 거래, 게임 경제 시스템

#### 플로이드-워셜 (Floyd-Warshall Algorithm)
- **특징**: 모든 노드 쌍 간의 최단거리 계산
- **시간복잡도**: O(V³)
- **공간복잡도**: O(V²)
- **활용**: 작은 규모 그래프의 완전 분석

---

### 3. 최소 신장 트리 (Minimum Spanning Tree, MST)

#### 크루스칼 알고리즘 (Kruskal's Algorithm)
- **접근방식**: 탐욕 알고리즘 (가중치가 작은 간선부터 선택)
- **자료구조**: Union-Find (Disjoint Set)
- **시간복잡도**: O(E log E)
- **활용**: 네트워크 설계, 전력망 구성

#### 프림 알고리즘 (Prim's Algorithm)
- **접근방식**: 탐욕 알고리즘 (한 노드에서 확장)
- **자료구조**: 우선순위 큐
- **시간복잡도**: O(V²) 또는 O((V + E) log V)
- **활용**: 그래프가 밀도 높을 때 효율적

---

### 4. 기타 중요 알고리즘

#### 위상정렬 (Topological Sort)
- **용도**: 방향 그래프에서 선형 순서 결정
- **조건**: DAG(Directed Acyclic Graph)에서만 가능
- **시간복잡도**: O(V + E)
- **활용**: 작업 스케줄링, 컴파일러 의존성 분석

#### 강연결 요소 (Strongly Connected Components, SCC)
- **용도**: 그래프의 강결정 부분 찾기
- **알고리즘**: Tarjan, Kosaraju
- **시간복잡도**: O(V + E)
- **활용**: 사이클 감지, 컴포넌트 분석

#### 이분 그래프 판정 (Bipartite Graph Check)
- **용도**: 그래프를 두 그룹으로 나눌 수 있는지 확인
- **방법**: BFS/DFS로 2-coloring 수행
- **시간복잡도**: O(V + E)
- **활용**: 체스판 칠하기, 매칭 문제

---

## 알고리즘 비교표

| 알고리즘 | 시간복잡도 | 용도 | 제약사항 |
|---------|----------|------|--------|
| BFS | O(V+E) | 최단거리(가중치 없음) | - |
| DFS | O(V+E) | 탐색, 백트래킹 | - |
| 다익스트라 | O((V+E)logV) | 최단거리 | 양수 가중치만 |
| 벨만-포드 | O(VE) | 최단거리 | 음수 사이클 없음 |
| 플로이드-워셜 | O(V³) | 모든 쌍 최단거리 | 작은 그래프 |
| 크루스칼 | O(ElogE) | MST | - |
| 프림 | O(V²) or O((V+E)logV) | MST | - |

## 실생활 활용 사례

- 🗺️ **네비게이션**: 지도 앱의 최단 경로 탐색 (다익스트라)
- 🌐 **소셜 네트워크**: 친구 추천, 관계도 분석 (BFS, SCC)
- 💻 **네트워크 라우팅**: 최적 경로 결정
- 📊 **작업 스케줄링**: 작업 간 의존성 관리 (위상정렬)
- 🔌 **전력망/통신망**: 최소 비용 연결 (MST)
- 🎮 **게임**: NPC 경로 설정, 상태 관리
- 🔍 **웹 크롤러**: 페이지 탐색 (BFS/DFS)

## 선택 기준

**어떤 알고리즘을 사용할까?**

1. **최단 경로를 찾아야 한다면?**
   - 가중치 없음 → BFS
   - 양수 가중치 → 다익스트라
   - 음수 가중치 → 벨만-포드
   - 모든 쌍 → 플로이드-워셜

2. **그래프 전체를 탐색해야 한다면?**
   - 모든 노드 방문 필요 → DFS/BFS
   - 재귀 선호 → DFS
   - 레벨별 처리 필요 → BFS

3. **연결 최소화를 원한다면?**
   - 간선이 적음 → 크루스칼
   - 노드가 적음 → 프림

## 참고자료

- 시간복잡도 분석의 중요성: 그래프 크기에 따라 알고리즘 선택이 크게 영향
- 자료구조 선택: 우선순위 큐, Union-Find 등 보조 자료구조가 성능 결정
- 전처리 단계: 그래프 표현 방식 선택 (인접 리스트 vs 인접 행렬)
