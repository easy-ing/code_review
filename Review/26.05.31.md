# **📌 사막의 선인장 구역 정하기**

**문제 유형**

2차원 배열, 슬라이딩 윈도우, 덱(Deque), 최솟값 관리

---

### 코드

```python
from collections import deque

def solution(m, n, h, w, drops):
    INF = len(drops) + 1

    grid = [[INF] * n for _ in range(m)]

    for time, (r, c) in enumerate(drops, start=1):
        grid[r][c] = time

    row_min = []

    for r in range(m):
        dq = deque()
        temp = []

        for c in range(n):
            while dq and grid[r][dq[-1]] >= grid[r][c]:
                dq.pop()

            dq.append(c)

            if dq[0] <= c - w:
                dq.popleft()

            if c >= w - 1:
                temp.append(grid[r][dq[0]])

        row_min.append(temp)

    best_time = -1
    answer = [0, 0]

    for c in range(n - w + 1):
        dq = deque()

        for r in range(m):
            while dq and row_min[dq[-1]][c] >= row_min[r][c]:
                dq.pop()

            dq.append(r)

            if dq[0] <= r - h:
                dq.popleft()

            if r >= h - 1:
                top = r - h + 1
                current_time = row_min[dq[0]][c]

                if current_time > best_time:
                    best_time = current_time
                    answer = [top, c]

    return answer
```

---

## **핵심 아이디어**

- 각 칸에 비가 떨어지는 시점을 저장
- 비가 떨어지지 않는 칸은 `INF`로 처리
- 선인장 구역이 처음 비를 맞는 시점은:
    - 해당 `h × w` 구역 안의 가장 빠른 빗방울 시점
    - 즉, 구역 내부의 최솟값
- 모든 `h × w` 구역의 최솟값 중 가장 큰 값을 선택
- 값이 같다면 더 위쪽, 더 왼쪽 좌표를 선택

---

## **시간 복잡도**

- 전체 격자 크기를 `m × n`이라고 할 때
- 각 칸은 덱에 최대 한 번 들어가고 한 번 나옴

따라서 전체 시간 복잡도는:

O(m × n)

---

## **총평**

이 문제는 단순히 모든 `h × w` 구역을 직접 확인하면 시간 초과가 발생할 수 있습니다.

따라서 행 방향으로 먼저 `w`칸 구간의 최솟값을 구하고, 그 결과를 다시 세로 방향으로 `h`칸씩 묶어 최솟값을 계산하는 방식이 적절합니다.

덱을 활용한 슬라이딩 윈도우 최솟값 풀이로, 큰 격자에서도 효율적으로 동작하는 좋은 접근입니다.
