# **📌예산**

그리디 / 정렬 / 배열

---

### 코드

```python
def solution(d, budget):
    d.sort()
    answer = 0

    for cost in d:
        if budget >= cost:
            budget -= cost
            answer += 1
        else:
            break

    return answer
```

---

### **💡 핵심 아이디어**

최대한 많은 부서를 지원해야 하므로,

**신청 금액이 작은 부서부터 지원**하면 됩니다.

예산은 한정되어 있기 때문에 큰 금액부터 지원하면

지원 가능한 부서 수가 줄어들 수 있습니다.

---

### **👍 잘한 점**

**1. 그리디 접근이 적절하다**

목표가 “지원 금액의 총합 최소화”가 아니라

“지원 가능한 부서 수 최대화”이므로 작은 금액부터 고르는 방식이 가장 적합합니다.

**2. 정렬 후 순회만 하면 된다**

배열을 오름차순 정렬한 뒤,

예산이 허용하는 동안 하나씩 차감하면 됩니다.

**3. 불필요한 탐색을 줄였다**

예산보다 큰 신청 금액을 만나면

그 이후 금액들은 더 크거나 같으므로 바로 break 할 수 있습니다.

---

### **⚠️ 아쉬운 점**

d.sort()는 원본 배열을 직접 변경합니다.

문제 풀이에서는 상관없지만, 원본 보존이 중요하다면 sorted()를 쓰는 편이 좋습니다.

### **✅ 총평**

이 문제는 **가장 적은 비용부터 선택해 개수를 최대화하는 전형적인 그리디 문제**입니다.

핵심은 다음입니다.

- 신청 금액 오름차순 정렬
- 예산에서 차감
- 지원 가능한 부서 수 카운트

전체적으로 보면

**정렬만 잘 떠올리면 쉽게 풀 수 있는 기본 그리디 문제**입니다.
