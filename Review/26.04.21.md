# **📌숫자의 표현**

### **문제 유형**

투 포인터 / 슬라이딩 윈도우 / 구현

---

### 코드

```python
def solution(n):
    answer = 0
    start = 1
    end = 1
    current_sum = 1

    while start <= n:
        if current_sum == n:
            answer += 1
            current_sum -= start
            start += 1
        elif current_sum < n:
            end += 1
            current_sum += end
        else:
            current_sum -= start
            start += 1

    return answ
```

---

### **💡 핵심 아이디어**

이 문제는 **연속된 자연수의 합**을 구하는 문제이므로,

매번 처음부터 끝까지 다 더해보는 것보다 **구간 합을 이동시키는 방식**이 더 효율적입니다.

즉,

- start부터 end까지의 합을 유지하면서
- 합이 n보다 작으면 end를 늘리고
- 합이 n보다 크면 start를 줄이는 식으로

하나의 연속 구간을 슬라이딩하듯 이동하면 됩니다.

예를 들어 n = 15일 때:

- 1 → 작음
- 1+2 → 작음
- 1+2+3+4+5 = 15 → 경우의 수 1개
- 이후 시작점을 옮기면서 계속 탐색

이 방식으로 모든 경우를 찾을 수 있습니다.

---

### **👍 잘한 점**

**1. 연속 구간 문제를 투 포인터로 잘 풀 수 있다**

연속된 수의 합이라는 점에서

이 문제는 전형적인 투 포인터 문제로 볼 수 있습니다.

**2. 중복 없이 모든 경우를 탐색할 수 있다**

각 구간은 start, end가 한 방향으로만 움직이기 때문에

불필요한 중복 계산이 없습니다.

**3. 시간 복잡도가 효율적이다**

start와 end가 각각 최대 n까지 한 번씩만 이동하므로

전체 시간 복잡도는 O(n) 수준입니다.

---

### **⚠️ 아쉬운 점**

이 문제를 처음 보면 이중 반복문으로

모든 시작점마다 합을 계속 더해보는 방식으로 풀기 쉽습니다.

### **✅ 총평**

이 문제는 **연속된 자연수의 합**이라는 점에서

슬라이딩 윈도우 / 투 포인터로 접근하기 좋은 대표 문제입니다.

핵심은 다음과 같습니다.

- 현재 구간 합이 작으면 범위를 늘리고
- 크면 시작점을 당기고
- 같으면 경우의 수를 세고 다음으로 이동

즉,

**연속 구간의 합을 실시간으로 조절하면서 탐색하는 문제**입니다.
