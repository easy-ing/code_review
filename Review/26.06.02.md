# **📌 리프 노드 수 최대화**

**문제 유형**

트리(Tree), 수학(Math), 완전 탐색, 그리디

---

### 코드

```python
def solution(dist_limit, split_limit):
    answer = 1

    pow2 = 1

    while pow2 <= split_limit:
        pow3 = 1

        while pow2 * pow3 <= split_limit:
            leaf = pow2 * pow3

            used = (pow2 - 1) + pow2 * ((pow3 - 1) // 2)

            if used <= dist_limit:
                answer = max(answer, leaf)

                remain = dist_limit - used

                for child in [2, 3]:
                    if leaf * child <= split_limit:
                        add = min(remain, leaf)
                        answer = max(answer, leaf + add * (child - 1))

            pow3 *= 3

        pow2 *= 2

    return answer
```

---

## **핵심 아이디어**

- 분배 노드는 자식 수가 `2개` 또는 `3개`
- 리프 수는 깊이가 늘어날 때마다:
    - `2배`
    - 또는 `3배`
- 분배도 제한 때문에 만들 수 있는 깊이는 많지 않음
    - `2^x * 3^y <= split_limit`
- 따라서 가능한 `2의 개수`, `3의 개수` 조합을 모두 확인
- 분배 노드 개수가 `dist_limit` 이하인 경우 중 리프 수 최댓값 선택

---

## **시간 복잡도**

- `split_limit`이 최대 `10^9`
- 가능한 `2^x`, `3^y` 조합만 확인

따라서 전체 시간 복잡도는:

O(log² split_limit)

---

## **총평**

이 문제는 실제 트리를 직접 만들면 복잡해지지만, 리프 수가 결국 `2`와 `3`을 몇 번 곱했는지로 결정된다는 점을 이용하면 훨씬 단순해집니다.

`2^x * 3^y` 조합을 탐색하면서 분배도 제한과 분배 노드 개수 제한을 동시에 확인한 풀이입니다.
