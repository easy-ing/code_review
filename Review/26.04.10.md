# **📌 대장균 3세대 찾기**

### **문제 유형**

MySQL - SQL (재귀 CTE, 계층형 데이터 탐색)

---

### 코드

```sql
WITH RECURSIVE ecoli_tree AS (
    SELECT 
        ID,
        PARENT_ID,
        1 AS generation
    FROM ECOLI_DATA
    WHERE PARENT_ID IS NULL

    UNION ALL

    SELECT 
        e.ID,
        e.PARENT_ID,
        t.generation + 1
    FROM ECOLI_DATA e
    JOIN ecoli_tree t
        ON e.PARENT_ID = t.ID
)
SELECT ID
FROM ecoli_tree
WHERE generation = 3
ORDER BY ID;
```

---

---

### **💡 핵심 아이디어**

대장균은 부모-자식 관계로 이어지는 **트리 구조**

→ 재귀 CTE를 이용해 **세대(generation)를 1씩 증가시키며 탐색**

→ 마지막에 generation = 3만 필터링하여 결과 출력

---

### **👍 잘한 점**

- 재귀 CTE를 사용해 계층 구조를 정확하게 모델링함
- generation 컬럼을 통해 세대 개념을 명확하게 표현
- 문제 확장(4세대, 전체 트리 등)에 대응 가능한 구조
- 초기 조건(1세대)과 재귀 조건이 분리되어 가독성이 좋음

---

### **🔍 아쉬운 점**

- 3세대까지만 필요한 문제인데 전체 트리를 끝까지 탐색함
- 재귀 깊이 제한이 없어 불필요한 연산이 발생할 수 있음
- 이 문제에 한정하면 3중 JOIN이 더 간단한 풀이가 될 수 있음

---

### **🧾 총평**

계층형 데이터를 재귀적으로 탐색하는 정석적인 풀이로,

문제 구조를 정확하게 이해한 좋은 접근이다.

다만 문제 요구가 고정되어 있는 경우에는

재귀 깊이 제한을 추가하거나 JOIN으로 단순화하면 더 효율적이다.

---

### **📚 배운 점**

- 계층형 데이터는 **WITH RECURSIVE**로 해결하는 것이 정석이다
- 트리 문제에서는 **세대(depth) 개념을 명확히 정의하는 것이 중요하다**
- 문제 요구 범위에 따라 **범용성 vs 단순성 선택이 필요하다**
- 재귀 쿼리는 반드시 **탐색 범위를 제한하는 습관**이 중요하다
