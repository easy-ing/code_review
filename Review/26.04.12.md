# **📌 트럭 대여 기록별 대여 금액 조회하기**

### **문제 유형**

SQL - JOIN, 조건 분기, 날짜 계산, 할인율 적용, 정렬

---

### 코드

```sql
SELECT
    H.HISTORY_ID,
    FLOOR(
        C.DAILY_FEE
        * (DATEDIFF(H.END_DATE, H.START_DATE) + 1)
        * (100 - IFNULL(P.DISCOUNT_RATE, 0)) / 100
    ) AS FEE
FROM CAR_RENTAL_COMPANY_RENTAL_HISTORY H
JOIN CAR_RENTAL_COMPANY_CAR C
    ON H.CAR_ID = C.CAR_ID
LEFT JOIN CAR_RENTAL_COMPANY_DISCOUNT_PLAN P
    ON C.CAR_TYPE = P.CAR_TYPE
   AND P.DURATION_TYPE = CASE
       WHEN DATEDIFF(H.END_DATE, H.START_DATE) + 1 >= 90 THEN '90일 이상'
       WHEN DATEDIFF(H.END_DATE, H.START_DATE) + 1 >= 30 THEN '30일 이상'
       WHEN DATEDIFF(H.END_DATE, H.START_DATE) + 1 >= 7 THEN '7일 이상'
       ELSE NULL
   END
WHERE C.CAR_TYPE = '트럭'
ORDER BY FEE DESC, H.HISTORY_ID DESC;
```

---

### **💡 핵심 아이디어**

이 코드는 트럭 차량의 대여 기록을 기준으로

각 기록의 **대여 일수**를 계산하고,

그 일수에 맞는 **할인 구간**을 찾아 할인율을 적용한 뒤

최종 대여 금액을 구하는 방식이다.

전체 흐름은 다음과 같다.

- 대여 기록과 차량 정보를 조인
- 차량 종류가 트럭인 데이터만 필터링
- DATEDIFF + 1로 실제 대여 일수 계산
- 대여 일수에 따라 7일 이상 / 30일 이상 / 90일 이상 구간 결정
- 할인 정책과 연결
- 할인율을 반영한 최종 금액 계산
- 금액 내림차순, 동일 금액이면 기록 ID 내림차순 정렬

즉, 이 문제는 단순 조회가 아니라

**날짜 계산 + 조건 분기 + 할인 정책 매핑 + 금액 산출**이 한 번에 들어간 문제다.

---

### **✅ 잘한 점**

**1. 요구사항의 흐름을 정확히 구현했다**

이 문제는 단순 조인 문제가 아니라,

대여 기간에 따라 할인 정책이 달라지기 때문에 계산 순서가 중요하다.

이 코드는 그 순서를 자연스럽게 잘 반영했다.

**2. 대여 일수 계산을 올바르게 했다**

DATEDIFF(H.END_DATE, H.START_DATE) + 1로 계산한 점이 좋다.

이 문제는 시작일과 종료일을 모두 포함해야 하므로 +1이 꼭 필요하다.

실무에서도 기간 계산에서 이 부분을 자주 실수한다.

**3. 할인 구간 매핑이 명확하다**

CASE 문으로

- 90일 이상
- 30일 이상
- 7일 이상
    
    순으로 조건을 나눈 것은 매우 적절하다.
    
    조건 순서도 큰 구간부터 검사하고 있어 논리적으로 안전하다.
    

**4. 할인 없는 경우까지 안정적으로 처리했다**

7일 미만은 할인 정책이 없는데,

이를 LEFT JOIN과 IFNULL(P.DISCOUNT_RATE, 0)로 처리한 점이 좋다.

이런 방식은 누락 데이터가 있을 수 있는 실무 환경에서도 많이 쓰는 안정적인 패턴이다.

**5. 정수 처리 요구사항을 잘 반영했다**

문제에서 정수 부분만 출력하라고 했기 때문에 FLOOR()를 사용했다.

출력 형식까지 신경 쓴 점이 좋다.

---

### **⚠️ 아쉬운 점**

**1. 같은 계산식이 여러 번 반복된다**

DATEDIFF(H.END_DATE, H.START_DATE) + 1가 여러 군데 반복되고 있다.

문제 풀이에서는 괜찮지만, 실무에서는 중복 계산은 가독성을 떨어뜨리고 유지보수를 어렵게 만든다.

예를 들어 대여 일수를 먼저 구하는 서브쿼리나 CTE를 만들면 더 읽기 쉬워진다.

**2. 할인 정책 조인 조건이 다소 빽빽하다**

LEFT JOIN 안에 CASE가 직접 들어가 있어서

처음 보는 사람은 조인 조건을 한 번에 이해하기 어려울 수 있다.

특히 SQL에 익숙하지 않은 사람에게는

“조인 조건인지, 계산 로직인지”가 한눈에 들어오지 않을 수 있다.

**3. 계산식의 의미가 한 줄에 몰려 있다**

최종 금액 계산식이 길어서

일일 요금 × 대여 일수 × 할인 적용 비율이라는 의미가 바로 드러나지 않는다.

실무에서는 중간 컬럼을 분리하면 리뷰와 디버깅이 쉬워진다.

---

### **🔍 실무 관점에서의 코멘트**

이 코드는 **코딩 테스트 정답 코드로는 충분히 좋고 정확한 풀이**다.

특히 다음 세 가지를 잘 처리했다는 점이 강점이다.

- 기간 계산 정확성
- 할인 구간 판별 정확성
- 할인 없는 경우 예외 처리

실무에서는 여기에 한 걸음 더 나아가

**중복 계산 제거**와 **가독성 개선**을 하면 더 좋은 쿼리가 된다.

예를 들어 대여 일수를 미리 구해두고,

그 값을 기준으로 할인 구간과 최종 금액을 계산하면

검증과 수정이 훨씬 쉬워진다.

즉, 이 코드는

**정답으로서 안정적이고, 실무에서는 약간의 구조화만 더하면 더 좋은 SQL**이라고 볼 수 있다.

### **✨ 개선 버전**

조금 더 읽기 쉽게 정리하면 이렇게 쓸 수 있다.

```sql
WITH TRUCK_HISTORY AS (
    SELECT
        H.HISTORY_ID,
        H.CAR_ID,
        C.CAR_TYPE,
        C.DAILY_FEE,
        DATEDIFF(H.END_DATE, H.START_DATE) + 1 AS RENT_DAYS
    FROM CAR_RENTAL_COMPANY_RENTAL_HISTORY H
    JOIN CAR_RENTAL_COMPANY_CAR C
        ON H.CAR_ID = C.CAR_ID
    WHERE C.CAR_TYPE = '트럭'
)
SELECT
    T.HISTORY_ID,
    FLOOR(
        T.DAILY_FEE * T.RENT_DAYS * (100 - IFNULL(P.DISCOUNT_RATE, 0)) / 100
    ) AS FEE
FROM TRUCK_HISTORY T
LEFT JOIN CAR_RENTAL_COMPANY_DISCOUNT_PLAN P
    ON T.CAR_TYPE = P.CAR_TYPE
   AND P.DURATION_TYPE = CASE
       WHEN T.RENT_DAYS >= 90 THEN '90일 이상'
       WHEN T.RENT_DAYS >= 30 THEN '30일 이상'
       WHEN T.RENT_DAYS >= 7 THEN '7일 이상'
       ELSE NULL
   END
ORDER BY FEE DESC, T.HISTORY_ID DESC;
```

이렇게 하면

- 대여 일수 계산
- 할인 정책 연결
- 금액 계산

이 단계가 더 분리되어 보여서 읽기 편하다.
