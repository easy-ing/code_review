# **📌 스킬트리**

**문제 유형**

문자열, 구현, 순서 검증, 스택/큐처럼 생각하기

---

### 코드

```python
def solution(skill, skill_trees):
    answer = 0

    for tree in skill_trees:
        learned = ""

        for s in tree:
            if s in skill:
                learned += s

        if skill[:len(learned)] == learned:
            answer += 1

    return answer
```

---

## **핵심 아이디어**

스킬트리에서 **선행 스킬에 포함된 문자만 뽑아낸 뒤**, 그 순서가 `skill`의 앞부분과 일치하는지 확인하면 된다.

---

## **시간 복잡도**

`skill_trees`의 길이는 최대 20,

각 스킬트리의 길이는 최대 26이다.

따라서 시간복잡도는 O(N × M)이다

---

## **총평**

이 문제는 선행 스킬 외의 문자는 신경 쓰지 않아도 된다.

따라서 스킬트리에서 `skill`에 포함된 문자만 남기고, 그 결과가 선행 스킬 순서의 앞부분과 같은지만 확인하면 된다.

핵심은 `"CBD"` 전체를 반드시 다 배워야 하는 게 아니라, 배운 선행 스킬들이 `"C"`, `"CB"`, `"CBD"`처럼 순서대로만 나오면 된다는 점이다.
