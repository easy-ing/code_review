# **📌 콜라문제**

**문제 유형**

구현, 반복문, 수학

---

### 코드

```python
def solution(a, b, n):
    answer = 0

    while n >= a:
        coke = (n // a) * b
        remain = n % a

        answer += coke
        n = remain + coke

    return answer
```

---

## **핵심 아이디어**

빈 병이 a개 이상 있을 때만 콜라로 교환할 수 있다.

한번 교환할 때마다

받는 콜라 수 = (현재 빈 병 수 // a) * b

그리고 남는 빈 병은 현재 빈 병 수 % a이다.

교환해서 받은 콜라도 마시면 다시 빈 병이 되므로, 다음 빈 병수는

남은 빈 병 + 받은 콜라수가 된다

---

## **시간 복잡도**

교환할 때마다 빈 병 수가 줄어들기 때문에 반복 횟수는 많지않다

대략적으로 O(log N)정도로 볼 수 있다.

---

## **총평**

이 문제는 **빈 병을 계속 콜라로 바꾸는 시뮬레이션 문제**이다.

핵심은 받은 콜라를 마시면 다시 빈 병이 된다는 점이다.

따라서 매번 n = 남은 빈 병 + 새로 받은 콜라 병으로 갱신 하면서,

받을 수 있는 콜라 수를 누적하면 된다.
