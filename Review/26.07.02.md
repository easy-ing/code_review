# **📌 카드뭉치**

**문제 유형**

큐, 구현, 그리디,

---

### 코드

```python
def solution(cards1, cards2, goal):
    idx1 = 0
    idx2 = 0

    for word in goal:
        if idx1 < len(cards1) and cards1[idx1] == word:
            idx1 += 1

        elif idx2 < len(cards2) and cards2[idx2] == word:
            idx2 += 1

        else:
            return "No"

    return "Yes"
```

---

## **핵심 아이디어**

`goal`의 단어를 앞에서부터 하나씩 확인한다.

각 단어는 반드시

- `cards1`의 맨 앞 단어이거나
- `cards2`의 맨 앞 단어

여야 사용할 수 있다.

중간 카드를 건너뛸 수 없기 때문이다.

---

## **시간 복잡도**

goal의 길이를 N이라고 하면, 한번만 순회 하므로

O(N)이다

---

## **총평**

이 문제는 카드 뭉치의 **순서를 유지해야 하는지**가 핵심이다.

카드를 건너뛸 수 없기 때문에 매번 각 카드 뭉치의 **맨 앞 카드**만 확인해야 한다.

따라서 `cards1`, `cards2`의 현재 위치를 각각 `idx1`, `idx2`로 관리하면서 `goal`을 순서대로 만들 수 있는지 확인하면 된다.
