# **📌 대충 만든 자판**

**문제 유형**

해시맵, 문자열 탐색, 구현

---

### 코드

```python
def solution(keymap, targets):
    answer = []
    key_dict = {}

    for key in keymap:
        for i in range(len(key)):
            char = key[i]
            count = i + 1

            if char not in key_dict:
                key_dict[char] = count
            else:
                key_dict[char] = min(key_dict[char], count)

    for target in targets:
        total = 0

        for char in target:
            if char not in key_dict:
                total = -1
                break

            total += key_dict[char]

        answer.append(total)

    return answer
```

---

## **핵심 아이디어**

이 문제의 핵심은 **각 문자를 입력하기 위해 필요한 최소 키 입력 횟수를 미리 저장하는 것**이다.

`keymap`을 순회하면서 각 문자에 대해

- 처음 등장한 문자라면 입력 횟수 저장
- 이미 등장한 문자라면 기존 값과 현재 입력 횟수 중 더 작은 값 저장을 수행한다.

이후 `targets`의 각 문자열을 순회하면서

- 모든 문자가 딕셔너리에 존재하면 입력 횟수를 누적
- 하나라도 존재하지 않으면 `1`

을 결과에 저장하면 된다.

---

## **시간 복잡도**

- 최소 입력횟수 : 최대 O(K)번 실행
- target 계산 : 최대 O(T)번 실행
- 최종 O(K+T)번 그렇다면 최대 20,000번

---

## **총평**

처음에는 문자마다 어느 키에 있는지 찾으려고 하면 매번 `keymap` 전체를 탐색하게 되어 비효율적일 수 있다.

하지만 **각 문자의 최소 입력 횟수를 미리 저장해 두는 전처리 방식**을 사용하면 이후에는 딕셔너리 조회만으로 결과를 구할 수 있다.

해시맵을 활용한 대표적인 문자열 탐색 문제이며, “최소값을 저장해두고 반복 조회한다”는 아이디어를 떠올리는 것이 핵심이다.
