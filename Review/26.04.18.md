# **📌최솟값 만들기**

### **문제 유형**

그리디 / 정렬 / 배열

---

### 코드

```python
def solution(A, B):
    A.sort()
    B.sort(reverse=True)

    answer = 0
    for i in range(len(A)):
        answer += A[i] * B[i]

```

---

### **💡 핵심 아이디어**

이 문제의 핵심은

**작은 수끼리 곱하는 것이 아니라, 한쪽의 작은 수와 다른 쪽의 큰 수를 곱해야 전체 합이 최소가 된다**는 점입니다.

즉,

- A는 오름차순 정렬
- B는 내림차순 정렬

한 뒤 같은 인덱스끼리 곱해서 더하면 최솟값이 됩니다.

예를 들어

- A = [1, 2, 4]
- B = [5, 4, 4]

처럼 두면

- 1 × 5
- 2 × 4
- 4 × 4

가 되어 합이 최소가 됩니다.

이건 큰 수끼리 만나지 못하게 분산시키는 방식이라고 보면 됩니다.

---

### **👍 잘한 점**

**1. 정렬을 활용한 그리디 접근이 정확하다**

이 문제는 모든 경우를 직접 비교하면 경우의 수가 너무 많아지는데,

정렬만으로 최적해를 만들 수 있다는 점을 잘 이용해야 합니다.

**2. 구현이 매우 단순하고 안정적이다**

정렬 후 같은 위치의 값만 곱해서 누적하면 되기 때문에

로직이 짧고 실수할 여지도 적습니다.

**3. 시간 복잡도도 충분히 효율적이다**

배열 길이가 최대 1000이므로

정렬 O(n log n) 후 순회 O(n)을 해도 충분히 빠릅니다.
