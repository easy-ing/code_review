# **연속된 부분 구간의 합 코드리뷰**

## **문제 요약**

수열에서 연속된 부분 구간의 합이 M으로 나누어떨어지는 구간의 개수를 구하는 문제다.

브루트포스로 모든 구간을 확인하면 구간 수가 너무 많아서 시간 초과가 발생한다.

따라서 이 문제는 **누적합 + 나머지 성질**을 이용해 풀어야 한다.

---

# **제출 코드**

```java
import sys

input = sys.stdin.readline

N, M = map(int, input().split())
arr = list(map(int, input().split()))

count = [0] * M
prefix = 0

count[0] = 1

for num in arr:
    prefix = (prefix + num) % M
    count[prefix] += 1

answer = 0
for c in count:
    answer += c * (c - 1) // 2

print(answer)
```

---

# **전체 평가**

이 코드는 문제의 핵심 아이디어를 정확히 반영했고,

입력 크기 N ≤ 10^6 에도 맞는 효율적인 풀이이다.

특히 다음 점이 좋다.

- 누적합을 끝까지 저장하지 않고 **현재 나머지만 관리**함
- 나머지별 개수를 세어 **조합 공식으로 정답 계산**
- 시간복잡도를 O(N + M)으로 잘 제한함

즉, 단순히 정답만 맞춘 수준이 아니라

**메모리와 시간까지 함께 고려한 안정적인 풀이**라고 볼 수 있다.

---

# **잘한 점**

## **1. 문제를 구간합이 아니라 “누적합의 나머지 비교”로 전환한 점**

즉, 문제를

- “모든 구간합 계산”
    
    이 아니라
    
- “같은 나머지를 가지는 누적합 쌍의 개수 세기”
    
    로 바꾼 것이 핵심이다.
    

이런 방식의 문제 변환은 알고리즘 문제 해결에서 매우 중요한 사고 방식이다.

---

## **2. 누적합 배열 전체를 저장하지 않은 점**

이 문제에서는 누적합 전체를 배열로 저장할 수도 있지만,

현재 코드는 그렇게 하지 않고 다음처럼 처리했다.

- 누적합 전체 배열이 필요 없음
- 메모리 사용량 감소
- 구현이 더 간결해짐

특히 N이 최대 1,000,000이라서,

불필요한 배열을 하나 더 만드는 것보다 이렇게 **필요한 정보만 유지하는 방식**이 더 좋다.

실무에서도 “다 저장한 뒤 나중에 쓴다”보다는

“필요한 값만 즉시 반영한다”는 습관이 중요하다.

---

## **3. 나머지 개수 세기를 조합으로 연결한 점**

---

# **아쉬운 점**

전체적으로는 좋은 코드지만, 리뷰 관점에서 보완할 수 있는 부분도 있다.

## **1. 변수명이 조금 더 의도를 드러내면 좋다**

- count → remainder_count
- prefix → prefix_mod
- answer → result

이렇게 바꾸면, 코드를 처음 보는 사람도

“아, 이건 누적합 자체가 아니라 누적합의 나머지를 저장하는 거구나”

라고 바로 이해할 수 있다.

특히 실무에서는 짧은 이름보다 **의미가 드러나는 이름**이 유지보수에 더 유리하다.

---

## **2. 핵심 아이디어에 대한 주석이 한 줄 정도 있으면 더 좋다**

이 코드는 짧고 깔끔하지만,

알고리즘을 처음 보는 사람은 왜 갑자기 조합을 쓰는지 이해하기 어려울 수 있다.

따라서 핵심 로직 위에 한 줄 정도 설명을 붙이면 좋다.

---

# **개선 버전 예시**

아래는 같은 로직을 조금 더 읽기 쉽게 다듬은 버전이다.

```java
import sys

input = sys.stdin.readline

N, M = map(int, input().split())
numbers = list(map(int, input().split()))

remainder_count = [0] * M
prefix_mod = 0
result = 0

# 합이 0인 초기 누적합 1개
remainder_count[0] = 1

for number in numbers:
    prefix_mod = (prefix_mod + number) % M
    remainder_count[prefix_mod] += 1

# 같은 나머지를 가진 누적합 2개를 고르는 경우의 수
for count in remainder_count:
    result += count * (count - 1) // 2

print(result)
```
