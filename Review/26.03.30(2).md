# **📌 조건에 맞게 수열 변환하기 3 코드 리뷰**

## **🧾 문제 요약**

정수 배열 arr와 정수 k가 주어졌을 때,

- k가 **홀수**이면 배열의 각 원소에 k를 **곱하고**
- k가 **짝수**이면 배열의 각 원소에 k를 **더한**

새로운 배열을 반환하는 문제이다.

---

## **💻 코드**

```java
class Solution {
    public int[] solution(int[] arr, int k) {
        int[] answer = new int[arr.length];
        for(int i = 0; i < arr.length; i++) {
            if(k % 2 != 0) {
                answer[i] = arr[i] * k;
            } else {
                answer[i] = arr[i] + k;
            }
        }
        return answer;
    }
}
```

---

## **🧠 핵심 로직**

```java
if(k % 2 != 0) {
    answer[i] = arr[i] * k;
} else {
    answer[i] = arr[i] + k;
}
```

👉 의미:

- k가 홀수면 **모든 원소에 곱셈 적용**
- k가 짝수면 **모든 원소에 덧셈 적용**

즉, **배열 원소마다 조건이 달라지는 것이 아니라 k의 상태에 따라 전체 배열에 동일한 연산을 적용**하는 구조이다.

---

## **⚙️ 시간복잡도 / 공간복잡도**

### **시간복잡도:**

### **O(n)**

- 배열을 한 번 순회하므로 선형 시간 복잡도

### **공간복잡도:**

### **O(n)**

- 결과를 저장하기 위한 새 배열 answer를 사용

---

## **✅ 잘한 점 (실무 기준)**

### **1. 원본 배열을 훼손하지 않음**

- 입력 배열 arr를 직접 수정하지 않고, 별도의 결과 배열을 생성함
- 사이드 이펙트를 줄이는 안전한 방식

### **2. 로직이 단순하고 직관적임**

- 문제 요구사항이 코드에 그대로 반영되어 있어 이해하기 쉽다.

### **3. 시간복잡도가 효율적임**

- 불필요한 중첩 반복문 없이 한 번의 순회로 해결

---

## **⚠️ 아쉬운 점 / 개선 포인트**

### **1. 같은 조건을 반복문 안에서 계속 검사하고 있음**

현재 코드는 배열의 모든 원소를 순회할 때마다 매번 k % 2 != 0을 검사한다.

```java
for(int i = 0; i < arr.length; i++) {
    if(k % 2 != 0) {
        ...
    } else {
        ...
    }
}
```

하지만 k의 홀짝 여부는 반복문이 도는 동안 바뀌지 않는다.

즉, **한 번만 판단하면 되는 조건을 매 반복마다 다시 검사하고 있는 셈**이다.

실행 성능 차이는 크지 않을 수 있지만,

실무에서는 이런 부분을 정리해두면 **가독성과 의도 전달 측면에서 더 좋다.**

---

## **✨ 개선된 코드 : 조건을 반복문 밖으로 분리**

```java
class Solution {
    public int[] solution(int[] arr, int k) {
        int[] result = new int[arr.length];

        if (k % 2 != 0) {
            for (int i = 0; i < arr.length; i++) {
                result[i] = arr[i] * k;
            }
        } else {
            for (int i = 0; i < arr.length; i++) {
                result[i] = arr[i] + k;
            }
        }

        return result;
    }
}
```

### **장점**

- k의 홀짝 판단을 한 번만 수행
- 반복문 내부가 더 단순해짐
- 조건이 “배열 전체에 적용되는 규칙”이라는 점이 더 잘 드러남
