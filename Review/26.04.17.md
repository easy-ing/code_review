# **📌JadenCase 문자열 만들기**

### **문제 유형**

문자열 / 구현 / 상태 관리

---

### 코드

```sql
def solution(s):
    answer = ''
    new_word = True  # 단어의 시작 여부

    for char in s:
        if char == ' ':
            answer += char
            new_word = True
        else:
            if new_word:
                answer += char.upper()
                new_word = False
            else:
                answer += char.lower()

    return answer
```

---

### **💡 핵심 아이디어**

이 문제의 핵심은 **공백을 기준으로 다음 문자가 단어의 시작인지 아닌지 관리하는 것**입니다.

보통 문자열 문제에서 split()을 먼저 떠올릴 수 있지만,

이 문제는 **공백이 연속해서 나올 수 있고, 원래 공백 개수를 그대로 유지해야 하므로**

문자열을 한 글자씩 직접 순회하는 방식이 더 안전합니다.

즉,

- 공백을 만나면 → 다음 문자는 새 단어의 시작
- 단어의 시작 문자면 → 대문자 처리
- 그 외 문자면 → 소문자 처리

이 흐름으로 해결한 코드입니다.

---

### **👍 잘한 점**

**1. 문제 조건을 정확히 반영했다**

이 문제에서 가장 중요한 함정은 연속 공백인데,

split()을 쓰지 않고 문자 단위로 순회해서 공백을 그대로 살린 점이 좋습니다.

**2. 상태값 하나로 깔끔하게 해결했다**

new_word라는 boolean 변수 하나로

“지금 문자가 단어의 첫 글자인가?”를 관리한 점이 직관적입니다.

**3. 예외 케이스까지 자연스럽게 처리된다**

- 맨 앞이 공백인 경우
- 공백이 여러 개 연속되는 경우
- 첫 문자가 숫자인 경우

이런 케이스도 별도 복잡한 처리 없이 잘 동작합니다.

예를 들어 "3people unFollowed me"의 경우

3은 upper()를 해도 그대로 3이라 문제 없이 처리되고,

뒤 문자인 people은 자동으로 소문자로 정리됩니다.

---

### **⚠️ 아쉬운 점**

**1. 문자열을 +=로 계속 이어붙이는 방식은 비효율적일 수 있다**

파이썬에서 문자열은 immutable이라 answer += char를 반복하면

문자열이 매번 새로 생성됩니다.

이 문제는 문자열 길이가 최대 200이라 실제로는 통과에 문제가 없지만,

실무나 더 큰 입력에서는 리스트에 담았다가 마지막에 ''.join() 하는 방식이 더 좋습니다.

---

### **✅ 총평**

문제의 핵심 조건인 **연속 공백 유지**를 정확히 이해하고,

이를 위해 문자 단위 순회 + 상태 관리 방식으로 잘 풀어낸 코드입니다.

특히 이 문제는 단순히 문자열 함수 몇 개 쓰는 문제가 아니라,

**공백을 포함한 원본 문자열 구조를 유지하면서 대소문자를 바꿔야 한다**는 점이 중요합니다.

그 부분을 정확히 짚은 풀이입니다.

다만 파이썬에서는 문자열 누적 방식이 비효율적일 수 있으므로,

같은 로직이라도 **리스트 + join 방식으로 개선하면 더 깔끔한 코드**가 됩니다.
