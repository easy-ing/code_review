# **📌 신고결과받기**

**문제 유형**

해시, 딕셔너리, 집합, 문자열 처리

---

### 코드

```python
def solution(id_list, report, k):
    report = set(report)

    reported_count = {user: 0 for user in id_list}
    reported_by_me = {user: [] for user in id_list}

    for r in report:
        reporter, reported = r.split()

        reported_count[reported] += 1
        reported_by_me[reporter].append(reported)

    answer = []

    for user in id_list:
        mail_count = 0

        for reported_user in reported_by_me[user]:
            if reported_count[reported_user] >= k:
                mail_count += 1

        answer.append(mail_count)

    return answer
```

---

## **핵심 아이디어**

이 문제에서 가장 중요한 조건은 이것이다.

같은 사람이 같은 유저를 여러 번 신고해도 1번으로 처리된다.

따라서 먼저 `report`를 `set`으로 바꿔서 **중복 신고를 제거**해야 한다.

그 다음에는 크게 두 가지를 기록하면 된다.

1. 각 유저가 몇 번 신고당했는지
2. 각 유저가 누구를 신고했는지

예를 들어 "muzi frodo” 인 경우

- `frodo`의 신고당한 횟수 +1
- `muzi`가 `frodo`를 신고했다고 기록

마지막으로 신고당한 횟수가 `k` 이상인 유저가 정지된다.

그 정지된 유저를 신고한 사람은 메일을 받는다.

---

## **시간 복잡도**

report의 길이를 R, 유저 수를 N이라고 하면

전체 시간 복잡도는 O(R)이다

---

## **총평**

이 문제는 단순 구현처럼 보이지만, 핵심은 **중복 신고 제거**이다.

중복 제거를 하지 않으면 두 번째 예시처럼

["ryan con", "ryan con", "ryan con"] 같은 경우를 틀리게 된다.

따라서 이 문제는

`set`으로 중복을 제거하고, `dict`로 신고 횟수와 신고 관계를 관리하는 문제

라고 보면 된다.
