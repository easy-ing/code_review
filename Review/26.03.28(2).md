# **☕ 카페 주문 금액 계산 - 코드리뷰**

## **📌 문제 이해**

주어진 주문 문자열 배열 order를 기반으로 총 결제 금액을 계산하는 문제이다.

- 아메리카노: 4500원
- 카페라떼: 5000원
- "anything" → 아메리카노로 처리
- "americano", "cafelatte" → 차가운 음료로 처리 (가격 동일)

👉 핵심: **결국 가격은 두 가지(4500 / 5000)만 존재**

---

## **💡 풀이 전략**

문자열을 하나씩 확인하면서:

- "cafelatte"가 포함되어 있으면 → 5000원
- 그 외 모든 경우 → 4500원

👉 다양한 문자열 케이스를 모두 나열하지 않고

👉 **가격 기준으로 단순화하여 해결**

---

## **💻 코드**

```java
class Solution {
    public int solution(String[] order) {
        int answer = 0;
        for(String o : order) {
            if(o.contains("cafelatte")) {
                answer += 5000;
            } else {
                answer += 4500;
            }
        }
        return answer;
    }
}
```

---

## **✅ 장점**

### **1. 문제를 본질적으로 단순화함**

- 문자열 종류는 많지만, 실제 가격은 2가지뿐
- "cafelatte" 포함 여부만으로 분기한 점이 핵심

👉 불필요한 조건 제거 → 코드 간결

---

### **2. 가독성이 매우 좋음**

- 반복문 + 조건문 구조가 직관적
- 로직을 한눈에 이해 가능

👉 유지보수 및 협업에 유리

---

### **3. 불필요한 비교 제거**

```java
o.equals("iceamericano") || o.equals("americanoice") ...
```

같은 긴 조건문 없이 해결

👉 실수 가능성 감소

---

### **4. 시간복잡도 효율적**

- O(n) (배열 1회 순회)
- 추가 공간 사용 없음 (O(1))

---

## **⚠️ 개선점**

### **1. 매직 넘버 사용**

```java
answer += 5000;
answer += 4500;
```

👉 의미가 드러나지 않음

✔ 개선:

```java
private static final int AMERICANO_PRICE = 4500;
private static final int LATTE_PRICE = 5000;
```

---

### **2. 변수명 개선 가능**

```java
for(String o : order)
```

✔ 개선:

```java
for(String menu : order)
```

---

### **3. 확장성 제한**

- 현재는 메뉴가 2개라 문제 없음
- 하지만 메뉴가 추가되면 contains() 방식은 위험할 수 있음

👉 문제 특화형 코드

---

## **🔧 리팩터링 코드**

```java
class Solution {

    private static final int AMERICANO_PRICE = 4500;
    private static final int LATTE_PRICE = 5000;

    public int solution(String[] order) {
        int totalPrice = 0;

        for (String menu : order) {
            if (menu.contains("cafelatte")) {
                totalPrice += LATTE_PRICE;
            } else {
                totalPrice += AMERICANO_PRICE;
            }
        }

        return totalPrice;
    }
}
```

---

## **🧠 핵심 인사이트**

👉 “모든 경우를 처리하려고 하지 말고, **결과 기준(가격)**으로 단순화하자”

- 문자열 종류 → 많음 ❌
- 가격 종류 → 2개 ✅

👉 문제를 단순화하면 코드가 짧아지고 실수도 줄어든다
