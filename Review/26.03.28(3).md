# **📌 코드 리뷰 - 주사위 점수 계산**

## **💻 코드**

```java
class Solution {
    public int solution(int a, int b) {
        if(a%2==1 && b%2==1) {
            return (a*a)+(b*b);
        }
        else if(a%2==1 || b%2==1) {
            return 2*(a+b);
        }

        return Math.abs(a-b);
    }
}
```

---

## **✏️ 문제 해결 아이디어**

- 두 수 a, b의 **홀짝 여부**에 따라 점수를 다르게 계산
- 조건을 3가지로 나누어 처리
1. 둘 다 홀수 → 제곱합
2. 하나만 홀수 → 합 × 2
3. 둘 다 짝수 → 차의 절댓값

→ if - else if - return 구조로 간단하게 구현

---

## **✅ 좋은 점**

### **1. 문제 조건을 그대로 코드로 표현**

- 문제의 분기 조건과 코드 흐름이 동일하여 이해하기 쉬움

```java
if(a%2==1 && b%2==1)
else if(a%2==1 || b%2==1)
```

---

### **2. 불필요한 중첩 없이 깔끔한 구조**

- if → else if → return 형태로 직선적인 흐름
- 가독성이 좋고 빠르게 읽힘

---

### **3. 마지막 분기를 간결하게 처리**

```java
return Math.abs(a-b);
```

- 불필요한 else 없이 바로 반환
- 코드 깊이가 줄어들어 깔끔함

---

### **4. 표준 라이브러리 활용**

- Math.abs() 사용으로 절댓값 처리
- 직접 구현보다 안전하고 가독성 좋음

---

## **⚠️ 아쉬운 점**

### **1. 동일한 조건식 반복**

```java
a%2==1
b%2==1
```

- 여러 번 반복되어 가독성이 떨어질 수 있음
- 조건의 의미가 바로 드러나지 않음

👉 개선 필요

---

### **2. 의도가 코드에서 직접 드러나지 않음**

```java
return (a*a)+(b*b);
```

- 왜 제곱하는지 코드만 보면 바로 이해하기 어려움
- 협업 코드에서는 의미 전달이 중요

---

### **3. 불필요한 괄호 사용**

```java
(a*a)+(b*b)
```

- 연산 우선순위상 괄호 없어도 충분
- 자바 스타일상 간결하게 쓰는 것이 일반적

---

## **🔧 개선 코드**

```java
class Solution {
    public int solution(int a, int b) {
        boolean aOdd = a % 2 == 1;
        boolean bOdd = b % 2 == 1;

        if (aOdd && bOdd) {
            return a * a + b * b;
        }

        if (aOdd || bOdd) {
            return 2 * (a + b);
        }

        return Math.abs(a - b);
    }
}
```

---

## **🧠 배운 점**

- 조건이 반복되면 **변수로 추출하는 것이 가독성에 유리**
- 짧은 코드보다 **의도가 명확한 코드가 더 좋은 코드**
- else를 줄이면 코드가 더 깔끔해질 수 있음
- 표준 라이브러리를 적극 활용하는 것이 좋음

---

## **📝 한 줄 정리**

**반복되는 조건을 변수로 분리하면 더 좋은 코드가 된다.**
