# **📌 그림 확대 코드 리뷰**

## **🧾 문제 요약**

문자열 배열 picture가 주어졌을 때,

각 문자를 가로로 k배 확대하고,

각 행도 세로로 k번 반복하여

최종적으로 전체 그림을 k배 확대하는 문제이다.

---

## **💻 코드**

```java
class Solution {
    public String[] solution(String[] picture, int k) {
        int row = picture.length;
        int col = picture[0].length();
        String[] answer = new String[row * k];
        for (int i=0; i<row; i++) {
            String tmp = "";
            for (int j=0; j<col; j++) {
                tmp += String.valueOf(picture[i].charAt(j)).repeat(k);
            }
            for (int h=0; h<k; h++) {
                answer[i*k+h] = tmp;
            }
        }
        return answer;
    }
}
```

---

## **🧠 핵심 아이디어**

이 코드는 확대를 두 단계로 처리한다.

### **1) 가로 확대**

- 각 문자를 repeat(k)로 반복

### **2) 세로 확대**

- 완성된 한 줄을 k번 복사

즉,

**“문자 단위 확대 → 행 단위 반복”** 구조이다.

## **⚙️ 시간복잡도 / 공간복잡도**

### **시간복잡도**

겉보기에는 각 문자마다 반복하고 각 행마다 복사하므로 대략 선형적으로 보이지만,

현재 구현은 문자열 누적 방식 때문에 실제로는 더 비효율적일 수 있다.

```java
tmp += ....
```

이 부분은 문자열이 매번 새로 생성되므로 누적 비용이 발생한다.

엄밀히 보면:

- 문자열 이어붙이기 누적 비용 때문에 **비효율 증가 가능**
- 실무 관점에서는 StringBuilder 사용이 권장된다.

---

### **공간복잡도**

- 결과 배열 answer 저장: **O(row * k)**
- 각 행의 확대 문자열 tmp 저장

---

## **✅ 잘한 점**

### **1. 문제 요구사항을 정확히 반영함**

- 가로 확대와 세로 확대를 모두 정확히 구현했다.

### **2. 로직 흐름이 직관적임**

- “한 줄 확대 → k번 복사” 구조가 이해하기 쉽다.

**⚠️ 아쉬운 점 / 개선 포인트**

**1. 입력 검증이 없음**

코딩테스트에서는 보통 생략하지만, 실무라면 아래와 같은 방어 코드도 고려할 수 있다.

- picture.length == 0
