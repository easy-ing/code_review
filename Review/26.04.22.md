# **📌피보나치 수**

### **문제 유형**

DP / 반복문 / 모듈러 연산

---

### 코드

```python
def solution(n):
    MOD = 1234567

    a, b = 0, 1  # F(0), F(1)

    for _ in range(2, n + 1):
        a, b = b, (a + b) % MOD

    return b
```

---

### **💡 핵심 아이디어**

이 문제의 핵심은 두 가지입니다.

1. **피보나치 점화식**
    - F(n) = F(n-1) + F(n-2)
2. **매 연산마다 나머지 연산 적용**
    - 숫자가 커지기 때문에 % 1234567을 계속 적용해야 함

---

### **👍 잘한 점**

**1. 공간을 O(1)로 줄였다**

일반적으로 DP 배열을 만들 수도 있지만,

이 문제는 이전 두 값만 필요하므로 변수 두 개로 충분합니다.

**2. 모듈러 연산을 중간에 적용했다**

피보나치 수는 급격히 커지기 때문에

마지막에 한 번만 %를 하면 이미 값이 너무 커집니다.

**3. 반복문으로 안정적으로 구현했다**

재귀로 풀 경우

- 시간 초과 (중복 계산)
- 스택 오버플로우

가 발생할 수 있습니다.

반면 반복문은 O(n)으로 안정적으로 동작합니다.

---

### **❌ 자주 틀리는 포인트**

- % 1234567을 마지막에만 적용하는 경우 → ❌
- 재귀 사용 → ❌ (시간 초과)
- DP 배열로 풀되 메모리 낭비 → 가능하지만 비효율

---

### **✅ 총평**

이 문제는 전형적인 **피보나치 + 모듈러 연산 문제**입니다.

핵심은 다음입니다.

- 이전 두 값만 사용해서 반복문으로 계산
- 매 연산마다 % 1234567 적용

즉,

**DP의 기본 개념 + 오버플로우 방지 처리**를 묻는 문제입니다.

전체적으로 보면

**기초지만 실무에서도 중요한 패턴을 포함한 좋은 문제**입니다.
