# 26.03.24

---

## **📌 문제 개요**

- **문제:** 문자열 str1이 str2의 부분 문자열인지 확인
- **조건:** 포함되면 1, 아니면 0 반환

---

## **💻 작성 코드**

```sql
class Solution {
    public int solution(String str1, String str2) {
        return str2.contains(str1) ? 1 : 0;
    }
}
```

---

## **✅ 핵심 로직 설명**

- str2.contains(str1)
    
    → str2 안에 str1이 포함되어 있는지 확인 (boolean 반환)
    
- 삼항 연산자 사용

```sql
조건 ? 참 : 거짓
```

- 최종 동작
    - 포함됨 → 1
    - 포함 안됨 → 0

---

## **👍 잘한 점**

### **1. 문제 의도를 정확하게 반영**

- “부분 문자열 여부 확인” → contains()로 정확히 해결
- 방향 (str2.contains(str1))도 올바르게 설정됨

---

### **2. 표준 라이브러리 활용**

- 직접 반복문 구현 대신 String.contains() 사용
- 코드가 짧고 명확함
- 실무에서도 권장되는 방식

---

### **3. 간결한 코드 구조**

- 불필요한 변수, 반복문 없음
- 한 줄로 핵심 로직 표현 → 가독성 좋음

---

## **⚠️ 아쉬운 점 (실무 기준)**

### **1. 삼항 연산자 vs if문**

- 현재 코드:

```sql
return str2.contains(str1) ? 1 : 0;
```

• 실무에서는 가독성을 위해 아래 형태를 선호하기도 함:

```sql
if (str2.contains(str1)) {
    return 1;
}
return 0;
```

---

### **2. 예외 처리 없음 (실무 관점)**

- 현재 문제에서는 필요 없음
- 하지만 실무에서는 null 체크 고려 가능

예:

```sql
if (str1 == null || str2 == null) {
    return 0;
}
```

---

## **🔧 개선 코드 (실무 스타일)**

```sql
class Solution {
    public int solution(String str1, String str2) {
        if (str1 == null || str2 == null) {
            return 0;
        }

        if (str2.contains(str1)) {
            return 1;
        }

        return 0;
    }
}
```

---

## **🧠 느낀 점 / 정리**

- 문자열 포함 여부는 직접 구현보다 **내장 메서드 활용이 더 효율적**
- 짧은 코드도 좋지만, **상황에 따라 가독성도 고려해야 함**
- 코딩테스트와 실무 코드는 **작성 기준이 조금 다름**
    - 코테 → 간결함
    - 실무 → 안정성 + 명확성

---

## **🏁 한줄 정리**

> Java의 contains()를 활용해 문제를 간결하고 정확하게 해결한 코드이며, 실무에서는 가독성과 예외 처리까지 고려하면 더 좋은 코드가 된다.
