# **📌 원형 마을 도둑질 문제**

### **문제 유형**

Python - 동적 계획법(DP), 원형 배열, 경우 분리

---

### 코드

```sql
def solution(money):
    n = len(money)

    def rob(arr):
        length = len(arr)
        if length == 1:
            return arr[0]

        dp = [0] * length
        dp[0] = arr[0]
        dp[1] = max(arr[0], arr[1])

        for i in range(2, length):
            dp[i] = max(dp[i - 1], dp[i - 2] + arr[i])

        return dp[-1]

    case1 = rob(money[:-1])  # 첫 집 포함, 마지막 집 제외
    case2 = rob(money[1:])   # 첫 집 제외, 마지막 집 포함 가능

    return max(case1, case2)
```

---

### **💡 핵심 아이디어**

이 문제의 핵심은 집들이 **일직선이 아니라 원형으로 연결되어 있다**는 점이다.

일반적인 도둑질 문제라면

인접한 집을 동시에 털 수 없으므로

```sql
dp[i] = max(dp[i-1], dp[i-2] + money[i])
```

공식으로 해결할 수 있다.

하지만 이 문제는 첫 번째 집과 마지막 집도 서로 인접하기 때문에,

둘을 동시에 선택할 수 없다.

그래서 문제를 한 번에 풀지 않고 **두 가지 경우로 나눠야 한다.**

- 첫 집을 터는 경우 → 마지막 집은 제외
- 첫 집을 털지 않는 경우 → 마지막 집 선택 가능

즉, 아래 두 개의 일자형 DP 문제로 바꿔서 푸는 방식이다.

- money[:-1]
- money[1:]

그리고 두 결과 중 더 큰 값을 정답으로 사용한다.

---

### **✅ 잘한 점**

**1. 원형 문제를 일자형 문제 2개로 잘 변환했다**

이 문제의 핵심은 원형 구조를 어떻게 처리하느냐인데,

첫 집 포함 / 첫 집 제외로 나눈 접근은 가장 정석적이고 안전한 방법이다.

**2. 점화식이 정확하다**

dp[i] = max(dp[i - 1], dp[i - 2] + arr[i])

이 식은 현재 집을 털지 않는 경우와 터는 경우를 정확히 비교하고 있다.

문제를 제대로 이해한 풀이이다.

**3. 보조 함수로 로직을 분리했다**

rob(arr) 함수로 일자형 집 털기 로직을 따로 빼서

중복 없이 재사용한 점이 좋다.

같은 DP 로직을 두 번 직접 쓰는 것보다 훨씬 깔끔하다.

**4. 시간 복잡도가 적절하다**

전체 길이가 최대 1,000,000이기 때문에

완전탐색은 불가능하다.

이 코드는 두 번의 선형 탐색만 수행하므로 O(N)으로 충분히 효율적이다.

---

### **⚠️ 아쉬운 점**

**1. 공간을 조금 더 줄일 수 있다**

현재 dp 배열을 따로 만들고 있어서 O(N) 공간을 사용한다.

하지만 이 문제는 사실 이전 두 값만 필요하므로

변수 두세 개만으로 O(1) 공간 최적화가 가능하다.

즉, 정답으로는 충분히 좋지만

입력 크기가 큰 문제인 만큼 메모리까지 의식하면 더 좋다.

**2. n 변수를 직접 활용하지 않는다**

처음에 n = len(money)를 선언했지만,

이후 실제 계산에는 거의 사용되지 않는다.

큰 문제는 아니지만, 사용하지 않는 변수는 제거하는 편이 깔끔하다.

**3. 슬라이싱이 추가 메모리를 사용한다**

money[:-1], money[1:]는 새 리스트를 만든다.

입력 크기가 최대 1,000,000이므로

실무나 성능 민감 상황에서는 이 부분도 아쉬울 수 있다.

문제 풀이에서는 통과 가능성이 높지만,

더 엄격하게 보면 인덱스 범위를 넘겨주는 방식이 메모리 면에서 더 낫다.
