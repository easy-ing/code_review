# **📌 동전 0 코드리뷰**

## **🧾 문제 유형**

- 그리디 Greedy

---

## **💻 핵심 아이디어**

- 동전이 **배수 관계**이므로 큰 단위 동전부터 최대한 사용하는 것이 항상 최적 → 뒤에서부터 순회하며 몫(K // coin)을 누적

---

## **✅ 잘한 점**

- 문제 조건(배수 관계)을 정확히 파악하고 DP 대신 그리디 적용
- 불필요한 연산 없이 단순 반복으로 해결 (O(N))
- 구현이 직관적이고 로직 흐름이 명확함

---

## **⚠️ 아쉬운 점**

- for i in range(...) 대신 reversed(coins) 사용 시 가독성 개선 가능
- K를 직접 변경하기보다 별도 변수 사용 시 더 안전한 코드 작성 가능
- 변수명(count)을 coin_count 등으로 명확히 하면 의도 전달이 더 좋음

---

## **🧠 배운 점**

- **문제 조건이 알고리즘 선택을 결정한다**
    
    → “동전이 배수 관계”라는 조건 하나로 DP가 아니라 그리디를 선택할 수 있음
    
    → 문제를 풀기 전에 **조건을 먼저 해석하는 습관**이 중요
    
- **그리디가 성립하는 이유를 이해해야 한다**
    
    → 단순히 “큰 것부터 쓰자”가 아니라
    
    → **항상 최적이 보장되는 구조인지 판단**해야 함
    
    → (배수 관계 → 작은 단위로 대체 가능 → 그리디 OK)
    
- **시간복잡도 줄이는 사고 방식**
    
    → DP로 가면 O(N*K)까지 갈 수 있는 문제를
    
    → 조건을 활용하면 O(N)으로 해결 가능
    
    → “더 단순하게 풀 수 없을까?”를 항상 고민해야 함
    
- **코드는 단순할수록 좋다**
    
    → 짧고 명확한 반복문 하나로 해결
    
    → 복잡한 로직보다 **문제 해석이 더 중요**
    

---

## **🏁 총평**

> 문제의 핵심 조건을 잘 활용한 **정석적인 그리디 풀이**이며, 성능과 구현 모두 깔끔하다. 실무 기준에서도 충분히 좋은 코드지만, 가독성과 변수 네이밍을 조금만 다듬으면 더 완성도 높은 코드가 된다.
