**문제 유형**

해시맵, 문자열 파싱, 구현

---

### 코드

```python
def solution(record):
    answer = []

    nickname = {}
    logs = []

    for r in record:
        data = r.split()
        action = data[0]
        uid = data[1]

        if action == "Enter":
            name = data[2]
            nickname[uid] = name
            logs.append((uid, "님이 들어왔습니다."))

        elif action == "Leave":
            logs.append((uid, "님이 나갔습니다."))

        elif action == "Change":
            name = data[2]
            nickname[uid] = name

    for uid, message in logs:
        answer.append(nickname[uid] + message)

    return answer
```

---

## **핵심 아이디어**

이 문제는 닉네임이 중간에 바뀌면 **이전 메시지의 닉네임도 전부 최종 닉네임으로 바뀐다**는 점이 핵심이다.

그래서 메시지를 바로 완성하지 않고,

1. 유저 아이디별 최종 닉네임을 저장
2. 입장/퇴장 기록만 따로 저장
3. 마지막에 최종 닉네임으로 메시지 생성

하는 방식으로 풀면 된다.

`Change`는 메시지에 출력되지 않으므로 닉네임만 갱신한다.

---

## **시간 복잡도**

record를 한번 순회하면서 닉네임과 로그를 저장한다.

이후 저장된 로그를 다시 한번 순회하면서 결과 메세지를 만든다.

따라서 전체 시간복잡도는 O(N)이다.

---

## **총평**

이 문제는 메시지를 처음부터 완성해버리면 닉네임 변경 시 기존 메시지를 전부 수정해야 해서 비효율적이다.

따라서 **유저 아이디를 기준으로 기록을 남기고, 마지막에 최종 닉네임을 붙이는 방식**이 핵심이다.

`uid`는 변하지 않는 고유값이고, 닉네임은 변할 수 있으므로 닉네임 기준이 아니라 유저 아이디 기준으로 처리해야 한다.
