# **📌세대별 자식이 없는 대장균 수 구하기**

### **문제 유형**

SQL / 재귀 CTE / 계층형 쿼리 / 집계 / NOT IN

---

### 코드

```sql
WITH RECURSIVE ECOLI_GENERATION AS (
    SELECT
        ID,
        PARENT_ID,
        1 AS GENERATION
    FROM ECOLI_DATA
    WHERE PARENT_ID IS NULL

    UNION ALL

    SELECT
        E.ID,
        E.PARENT_ID,
        EG.GENERATION + 1 AS GENERATION
    FROM ECOLI_DATA E
    JOIN ECOLI_GENERATION EG
        ON E.PARENT_ID = EG.ID
)

SELECT
    COUNT(*) AS COUNT,
    GENERATION
FROM ECOLI_GENERATION
WHERE ID NOT IN (
    SELECT PARENT_ID
    FROM ECOLI_DATA
    WHERE PARENT_ID IS NOT NULL
)
GROUP BY GENERATION
ORDER BY GENERATION;
```

---

### **💡 💡 핵심 아이디어**

이 문제는 크게 두 단계로 나뉩니다.

1. **각 개체의 세대(GENERATION)를 구한다**
2. **그중 자식이 없는 개체만 골라서 세대별로 개수를 센다**

---

### **1️⃣ 세대 계산 (재귀 CTE)**

- PARENT_ID IS NULL → 1세대
- 이후 부모를 따라가면서 GENERATION + 1

즉 트리 구조를 따라 내려가면서

각 노드(개체)의 깊이를 구하는 문제입니다.

---

### **2️⃣ 자식이 없는 개체 판별**

자식이 없는 개체 =

**다른 행의 PARENT_ID로 한 번도 등장하지 않는 ID**

---

### **👍 잘한 점**

**1. 계층 구조를 재귀 CTE로 정확하게 해결했다**

이 문제는 단순 GROUP BY가 아니라

부모-자식 관계를 따라 내려가는 구조라서

재귀 CTE를 사용하는 게 정석입니다.

---

**2. 세대 계산 로직이 명확하다**

---

---

**3. “자식 없는 개체” 정의를 정확히 잡았다**

이 문제에서 핵심 함정은 이 부분입니다.

- 자식이 없는 개체 = leaf 노드
- 즉 **PARENT_ID로 등장하지 않는 ID**

이걸 정확히 이해하고 구현한 점이 좋습니다.

---

### **⚠️ 아쉬운 점**

**1. NOT IN은 NULL 이슈가 있을 수 있다**

---

**2. CTE 결과를 한 번 더 감싸서 쓰면 가독성이 좋아진다**

지금은 바로 WHERE에서 필터링했지만,

중간에 한 번 나누면 더 명확해집니다.

---

### **✅ 총평**

이 문제는 단순 집계 문제가 아니라

**트리 구조(계층형 데이터)를 이해하고 처리할 수 있는지**를 묻는 문제입니다.

핵심은 다음 두 가지입니다.

- 재귀 CTE로 세대(GENERATION) 계산
- 자식이 없는 노드(leaf node) 필터링

즉,

**계층 구조 + 집계 조건을 결합하는 SQL 문제**입니다.
