# **📌년, 월별 상품 구매 회원 수 구하기**

### **문제 유형**

SQL / JOIN / 집계 / 서브쿼리 / DISTINCT / 비율 계산

---

### 코드

```sql
SELECT
    YEAR(O.SALES_DATE) AS YEAR,
    MONTH(O.SALES_DATE) AS MONTH,
    COUNT(DISTINCT O.USER_ID) AS PURCHASED_USERS,
    ROUND(
        COUNT(DISTINCT O.USER_ID) / (
            SELECT COUNT(*)
            FROM USER_INFO
            WHERE YEAR(JOINED) = 2021
        ),
        1
    ) AS PURCHASED_RATIO
FROM USER_INFO U
JOIN ONLINE_SALE O
    ON U.USER_ID = O.USER_ID
WHERE YEAR(U.JOINED) = 2021
GROUP BY
    YEAR(O.SALES_DATE),
    MONTH(O.SALES_DATE)
ORDER BY
    YEAR,
    MONTH;
```

---

### **💡 핵심 아이디어**

이 문제의 핵심은 두 가지입니다.

첫째, **2021년에 가입한 회원만 먼저 대상으로 제한**해야 합니다.

둘째, 그 회원들 중에서 **각 년/월별로 실제 구매한 회원 수를 중복 없이 세고**,

이를 **2021년 가입 전체 회원 수로 나눠 비율을 구해야** 합니다.

즉 흐름은 다음과 같습니다.

- USER_INFO에서 2021년 가입 회원 추리기
- 그 회원과 ONLINE_SALE 연결하기
- 판매일 기준으로 년/월 묶기
- 월별 구매 회원 수는 COUNT(DISTINCT USER_ID)로 계산
- 비율은 월별 구매 회원 수 / 2021년 가입 전체 회원 수

---

### **👍 잘한 점**

**1. JOIN 방향이 명확하다**

이 문제는 구매 정보가 ONLINE_SALE에 있고,

가입 연도 조건은 USER_INFO에 있으므로

두 테이블을 USER_ID로 조인하는 것이 정석입니다.

---

**2. 중복 구매를 막기 위해 DISTINCT가 꼭 필요하다**

한 회원이 같은 달에 여러 번 구매할 수 있으므로

단순 COUNT(*)를 쓰면 구매 건수가 세어져서 오답이 됩니다.

그래서 반드시 처럼 **회원 수 기준**으로 세야 합니다.

---

**3. 비율 분모를 전체 2021년 가입자 수로 고정했다**

비율은 매달 달라지는 구매자 수를

항상 같은 기준인 “2021년 가입 전체 회원 수”로 나눠야 하므로,

분모를 서브쿼리로 고정하는 방식이 깔끔합니다.

---

### **⚠️ 아쉬운 점 / 주의할 점**

**1. COUNT(*)를 쓰면 틀린다**

이 문제는 “상품을 구매한 회원수”이지

“판매 건수”가 아닙니다.

예를 들어 같은 회원이 한 달에 2번 구매하면

회원 수는 1명인데 판매 건수는 2건이므로

반드시 DISTINCT USER_ID가 필요합니다.

---

**2. 판매일 기준으로 그룹화해야 한다**

가입일 기준이 아니라

실제 구매가 일어난 SALES_DATE의 년, 월로 묶어야 합니다.

---

**3. 반올림 자릿수 확인**

문제 예시 결과가 0.5, 0.3 형태이므로

소수 둘째 자리에서 반올림 = **소수 첫째 자리까지 출력**입니다.

---

### **✅ 총평**

이 문제는 단순 집계가 아니라

**조건에 맞는 회원 집합을 먼저 정한 뒤, 그 집합의 월별 구매 비율을 구하는 문제**입니다.

핵심은 다음 세 가지입니다.

- 2021년 가입 회원만 대상으로 제한
- 월별 구매 회원 수는 COUNT(DISTINCT USER_ID)
- 비율 분모는 2021년 가입 전체 회원 수

즉,

**JOIN + DISTINCT 집계 + 고정 분모 비율 계산**이 핵심인 문제입니다.

전체적으로 보면

**문제 조건을 정확히 읽고 회원 수와 판매 건수를 구분하는 것이 중요한 SQL 문제**입니다.
