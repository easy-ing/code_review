# **📌올바른 괄호**

### **문제 유형**

문자열 / 스택 / 구현

---

### 코드

```python
def solution(s):
    count = 0

    for ch in s:
        if ch == '(':
            count += 1
        else:
            count -= 1

        if count < 0:
            return False

    return count == 0
```

---

### **💡 핵심 아이디어**

이 문제는 실제로 스택을 써도 되지만,

( 와 ) 두 종류만 나오기 때문에 **열린 괄호의 개수만 세는 방식**으로 더 간단하게 풀 수 있습니다.

문자열을 왼쪽부터 보면서

- ( 이 나오면 열린 괄호 수 증가
- ) 이 나오면 열린 괄호 수 감소

시키면 됩니다.

다만 중요한 점은,

중간에라도 열린 괄호 수가 음수가 되면 안 됩니다.

왜냐하면 그건 곧

닫는 괄호 )가 여는 괄호 (보다 먼저 나왔다는 뜻이기 때문입니다.

예를 들어 ")()(" 는 첫 글자부터 ) 이므로 바로 올바르지 않은 괄호입니다.

---

### **👍 잘한 점**

**1. 불필요한 자료구조 없이 해결할 수 있는 문제라는 점을 잘 활용할 수 있다**

이 문제는 괄호 종류가 하나뿐이라

굳이 스택에 문자 자체를 저장할 필요가 없습니다.

개수만 관리해도 충분합니다.

**2. 중간에 바로 실패 처리할 수 있다**

순회 도중 count < 0 이 되는 순간 바로 False를 반환하므로

끝까지 볼 필요 없는 경우 빠르게 종료할 수 있습니다.

**3. 시간 효율이 매우 좋다**

문자열 길이가 최대 100,000이므로

한 번만 순회하는 O(n) 풀이가 적절합니다.

---

### **⚠️ 아쉬운 점**

초기 코드처럼 answer = True를 두고 마지막에 무조건 True를 반환하면

검증 로직이 전혀 들어가지 않아서 문제를 해결하지 못합니다.

이 문제는 단순 출력이 아니라

문자열 전체의 균형과 순서를 함께 확인해야 합니다.

또 하나 주의할 점은

마지막에 count == 0인지 반드시 확인해야 한다는 것입니다.

예를 들어 "(((" 은 중간에 음수가 되지 않지만

열린 괄호가 남아 있으므로 올바른 괄호가 아닙니다.

---

### **✅ 총평**

이 문제는 **스택의 기본 개념을 묻는 대표 문제**이지만,

입력 문자가 ( 와 ) 두 개뿐이라면

스택 대신 카운터 하나로 더 간단하게 해결할 수 있습니다.

핵심은 두 가지입니다.

- 순회 중 count가 음수가 되면 바로 실패
- 순회가 끝난 뒤 count가 0이어야 성공

즉,

**개수의 균형 + 순서의 올바름**을 함께 확인해야 하는 문제입니다.

전체적으로 보면

**매우 정석적이고 효율적인 풀이**입니다.
