# **📌 혼자 놀기의 달인**

**문제 유형**

그래프 탐색, 사이클 찾기, 방문 처리, 구현

---

### 코드

```python
def solution(cards):
    visited = [False] * len(cards)
    groups = []

    for i in range(len(cards)):
        if not visited[i]:
            count = 0
            current = i

            while not visited[current]:
                visited[current] = True
                count += 1
                current = cards[current] - 1

            groups.append(count)

    groups.sort(reverse=True)

    if len(groups) < 2:
        return 0

    return groups[0] * groups[1]
```

---

## **핵심 아이디어**

이 문제는 카드 번호를 따라가다 보면 결국 **사이클**이 생기는 구조다.

예를 들어 `cards[i]`는`i + 1번 상자 안에 적힌 다음 상자 번호`를 의미한다.

즉, 현재 상자 번호 → cards[현재 상자 번호 - 1] 로 계속 이동하면 된다.

각 사이클의 크기가 곧 하나의 그룹 크기다.

모든 상자를 돌면서 방문하지 않은 상자에서 탐색을 시작하고,

그 그룹의 크기를 구해 리스트에 저장한다.

마지막에는 그룹 크기 중 가장 큰 2개를 골라 곱하면 최고 점수다.

그룹이 1개뿐이면 두 번째 그룹을 만들 수 없으므로 `0`을 return 한다.

---

## **시간 복잡도**

각 상자는 한 번만 방문한다.

전체적으로 보면 cards의 모든 원소를 한 번씩 방문하므로 O(N)

그룹 크기 리스트를 정렬한다. 그럼 그룹 개수를 G라고 하면 O(G log G)

따라서 전체 시간 복잡도는 O(N + G log G)

---

## **총평**

실제로는 카드가 가리키는 번호를 따라가는 **사이클 그룹 문제**다.

각 그룹의 크기만 알면 되고, 최고 점수는 가장 큰 그룹 2개의 곱이다.

핵심은 `cards[current] - 1`이다.

카드 번호는 1번부터 시작하지만, 파이썬 인덱스는 0번부터 시작하므로 반드시 `-1` 처리가 필요하다.
