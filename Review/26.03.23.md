# 26.03.23

**문제 풀이 코드**

```sql
import java.util.*;

class Solution {
    public int[] solution(int[] arr, int[] delete_list) {
        List<Integer> list = new ArrayList<>();

        for (int n : arr) {
            list.add(n);
        }

        for (int n : delete_list) {
            list.remove((Integer) n);
        }

        int[] answer = new int[list.size()];
        for (int i = 0; i < list.size(); i++) {
            answer[i] = list.get(i);
        }

        return answer;
    }
}
```

---

## **코드 설명**

- arr의 원소를 ArrayList로 옮긴 뒤
- delete_list에 있는 값을 하나씩 삭제하고
- 남은 값을 다시 int[] 배열로 변환해서 반환하는 방식이다.

---

## **장점**

### **1. 로직이 직관적이다**

- 배열 → 리스트 변환 → 값 삭제 → 배열 변환 순서가 명확하다.
- 처음 보는 사람도 이해하기 쉽다.

### **2. 구현이 간단하다**

- 복잡한 자료구조 없이 ArrayList만으로 해결했다.
- 문제 요구사항을 빠르게 구현하기 좋다.

### **3.**

### **remove((Integer)n)**

### **를 정확히 사용했다**

- remove(int index)와 remove(Object o)의 차이를 이해하고
- 인덱스 삭제가 아니라 값 삭제가 되도록 처리했다.
- 자바 컬렉션 오버로딩을 정확히 이해한 코드라는 점이 좋다.

---

## **단점**

### **1. 성능이 비효율적일 수 있다**

- ArrayList에서 중간 원소를 삭제하면 뒤 원소들을 한 칸씩 당겨야 한다.
- delete_list 크기가 커질수록 반복 삭제 비용이 커진다.

### **2. 불필요한 변환 과정이 있다**

- int[] → List<Integer> → int[]로 두 번 변환한다.
- 문제는 해결되지만 메모리와 연산 측면에서 다소 비효율적이다.

### **3. 큰 입력에서는 최적 풀이가 아니다**

- 삭제 대상이 많을 경우 Set을 사용해 삭제 여부를 먼저 저장하고
- arr를 한 번만 순회하는 방식이 더 효율적이다.

---

## **개선 아이디어**

- delete_list를 HashSet에 저장한 뒤
- arr를 순회하면서 Set에 없는 값만 결과 리스트에 추가하면 더 효율적이다.

```sql
import java.util.*;

class Solution {
    public int[] solution(int[] arr, int[] delete_list) {
        Set<Integer> deleteSet = new HashSet<>();
        for (int n : delete_list) {
            deleteSet.add(n);
        }

        List<Integer> list = new ArrayList<>();
        for (int n : arr) {
            if (!deleteSet.contains(n)) {
                list.add(n);
            }
        }

        int[] answer = new int[list.size()];
        for (int i = 0; i < list.size(); i++) {
            answer[i] = list.get(i);
        }

        return answer;
    }
}
```
