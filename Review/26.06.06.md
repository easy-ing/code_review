# **📌 빛의 경로 사이클**

**문제 유형**

그래프 탐색, 시뮬레이션, 방향처리, 방문체크

---

### 코드

```python
def solution(grid):
    answer = []

    rows = len(grid)
    cols = len(grid[0])

    directions = [
        (-1, 0),  # 위
        (0, 1),   # 오른쪽
        (1, 0),   # 아래
        (0, -1)   # 왼쪽
    ]

    visited = [[[False] * 4 for _ in range(cols)] for _ in range(rows)]

    for r in range(rows):
        for c in range(cols):
            for d in range(4):
                if not visited[r][c][d]:
                    count = 0

                    while not visited[r][c][d]:
                        visited[r][c][d] = True
                        count += 1

                        if grid[r][c] == "L":
                            d = (d - 1) % 4
                        elif grid[r][c] == "R":
                            d = (d + 1) % 4

                        r = (r + directions[d][0]) % rows
                        c = (c + directions[d][1]) % cols

                    answer.append(count)

    answer.sort()
    return answer
```

---

## **핵심 아이디어**

- 빛의 상태는 단순히 위치만으로 결정되지 않음
- 같은 칸에 있어도 어느 방향으로 들어왔는지가 다르면 다른 경로가 될 수 있음
- 따라서 방문 체크는 다음 3가지 기준으로 해야 함
    - 행 위치 `r`
    - 열 위치 `c`
    - 방향 `d`

---

## **시간 복잡도**

- 격자의 크기를 `rows × cols`라고 할 때
- 각 칸마다 방향이 4개 존재
- 모든 상태는 한 번씩만 방문됨

따라서 전체 시간 복잡도는:

**O(rows × cols × 4)**

즉, 상수 4를 제외하면:

**O(rows × cols)**

---

## **총평**

이 문제는 단순한 위치 방문 문제가 아니라, 위치와 방향을 함께 상태로 관리해야 하는 시뮬레이션 문제입니다.

`visited[r][c][d]` 구조를 사용하여 모든 가능한 빛의 상태를 정확하게 추적한 점이 핵심입니다.

또한 `%` 연산자를 이용해 격자의 끝을 넘어갔을 때 반대편으로 이동하는 조건을 깔끔하게 처리했습니다.
