# **꼬리 문자열 코드리뷰**

## **문제**

문자열 리스트 str_list에서 특정 문자열 ex를 포함한 원소는 제외하고,

나머지 문자열을 순서대로 이어붙여 반환하는 문제.

예시:

- ["abc", "def", "ghi"], "ef" → "abcghi"

---

## **작성한 코드**

```sql
class Solution {
    public String solution(String[] str_list, String ex) {
        String answer = "";
        for (int i = 0; i < str_list.length; i++) {
            if (str_list[i].contains(ex)) {
                str_list[i] = "";
            }
            answer += str_list[i];
        }
        return answer;
    }
}
```

---

## **코드 해석**

- 문자열 결과를 저장할 answer를 빈 문자열로 초기화
- 배열을 처음부터 끝까지 순회
- 현재 문자열이 ex를 포함하면 ""로 변경
- 이후 해당 값을 answer에 이어붙임
- 최종적으로 완성된 문자열 반환

---

## **잘한 점**

### **1. 문제 요구사항을 정확히 구현했다**

- contains()를 사용해서 포함 여부를 잘 판단했다.
- 제외할 문자열과 포함할 문자열을 구분해 최종 결과를 만들었다.

### **2. 한 번의 반복으로 해결했다**

- 배열을 한 번만 순회하면서 처리했기 때문에 흐름이 단순하고 이해하기 쉽다.

### **3. 전체 로직이 직관적이다**

- 초보자가 보기에도 “확인 → 제외 처리 → 이어붙이기” 흐름이 눈에 잘 들어온다.

---

## **아쉬운 점**

### **1. 입력 배열**

### **str_list를 직접 수정하고 있다**

```sql
if (str_list[i].contains(ex)) {
    str_list[i] = "";
}if (str_list[i].contains(ex)) {
    str_list[i] = "";
}
```

- 이 부분은 원본 배열 값을 바꾸는 방식이다.
- 문제 풀이에서는 가능하지만, 실무에서는 보통 입력값을 직접 수정하지 않는 편이 좋다.
- 이유는 원본 데이터가 이후 다른 곳에서도 사용될 수 있기 때문이다.

즉, **정답은 맞지만 부작용(side effect)이 생길 수 있는 방식**이다.

---

### **2. 문자열 누적에 += 를 사용했다**

```sql
answer += str_list[i];
```

- 자바의 문자열은 불변 객체이기 때문에 +=를 반복하면 새로운 문자열 객체가 계속 생성될 수 있다.
- 이 문제는 데이터 크기가 작아서 큰 문제는 없지만,
- 실무나 면접에서는 보통 StringBuilder를 더 좋은 방식으로 본다.

---

### **3. 제외 로직이 조금 우회적이다**

현재 코드는

- 포함하면 빈 문자열로 바꾸고
- 그 후에 무조건 더하는 방식이다.

하지만 더 자연스러운 방식은

- **포함하지 않을 때만 결과에 추가하는 것**이다.

즉, 현재 코드도 동작은 맞지만 의도를 더 직접적으로 표현할 수 있다.

---

## **개선 방향**

### **개선 포인트**

- 원본 배열은 수정하지 않기
- 문자열 누적은 StringBuilder 사용하기
- 조건을 더 직접적으로 표현하기

---

## **개선 코드**

```sql
class Solution {
    public String solution(String[] str_list, String ex) {
        StringBuilder answer = new StringBuilder();

        for (int i = 0; i < str_list.length; i++) {
            if (!str_list[i].contains(ex)) {
                answer.append(str_list[i]);
            }
        }

        return answer.toString();
    }
}
```

## **개선 코드 설명**

- StringBuilder를 사용해서 문자열을 효율적으로 이어붙였다.
- ex를 포함하지 않는 경우에만 append() 했다.
- 입력 배열 str_list는 변경하지 않았다.
- 문제의 의도가 더 직접적으로 드러난다.
